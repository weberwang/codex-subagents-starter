# Owen

Owen 是一个面向 Codex 的多子代理编排插件骨架。

当前目录先提供最小可验证入口壳，目标是把后续的规划、取证、实施和复核能力收束到同一个插件入口下，避免子代理职责分散。

## 当前内容

- `.codex-plugin/plugin.json`：插件清单基线，字段对齐后续校验要求
- `skills/orchestrator/SKILL.md`：统一入口 skill，负责说明什么时候该进入编排流程

## 入口定位

`owen:orchestrator` 只负责把任务导向合适的编排链路，不替代底层子代理本身的职责：

- 需要先研究资料时，优先衔接 `librarian`
- 需要关键判断或方案取舍时，优先衔接 `oracle`
- 需要小范围落地实现时，交给 `implementer`
- 需要交付前阻塞复核时，交给 `momus`

后续如果补充运行资产，应继续围绕这个统一入口扩展，而不是让插件内部再出现职责重叠的第二套入口。

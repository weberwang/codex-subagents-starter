# Remove Dead Code

## 适用场景

- 发现明显未使用函数、文件、分支或配置
- 重构后需要清理遗留代码
- 需要降低维护噪音和阅读负担

## 执行步骤

1. 先定位疑似死代码。
2. 检查直接引用、间接引用和动态引用风险。
3. 只删除证据充分的未使用代码。
4. 删除后运行相关验证命令。
5. 汇总删除范围与验证结果。

## 边界约束

- 不把清理死代码扩展成大规模重构。
- 对用途不明或可能被动态调用的代码保持谨慎。
- 在引用关系不清时优先保留，并说明原因。

## 输出要求

- 明确指出删除了什么。
- 明确指出保留了什么以及保留原因。
- 附带删除后的验证结果。

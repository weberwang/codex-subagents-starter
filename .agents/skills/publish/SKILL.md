# Publish

## 适用场景

- 准备发布当前分支或当前版本
- 需要在推送或创建发布前做最后核对
- 需要梳理这次发布包含了哪些内容

## 执行步骤

1. 检查当前工作区是否干净。
2. 检查最近一次验证是否真实执行且结果明确。
3. 汇总本次发布涉及的主要变更内容。
4. 确认是否存在未完成项、已知风险或待说明限制。
5. 在一切明确后，再继续执行推送、打标签或创建发布动作。

## 边界约束

- 不自动决定版本号策略。
- 不代替平台侧发布流程。
- 如果工作区不干净或验证缺失，优先停止并指出阻塞点。

## 输出要求

- 给出发布前检查结果。
- 列出阻塞项或放行结论。
- 如果允许继续，明确建议下一步操作。

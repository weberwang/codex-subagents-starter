# GitHub Triage

## 适用场景

- 需要整理 issue 或 PR 队列
- 需要做 backlog 初步分流
- 需要给团队一个可执行的优先级建议

## 执行步骤

1. 汇总当前待处理项。
2. 按类型、紧急程度、影响范围进行归类。
3. 区分应立即处理、应延后处理和信息不足项。
4. 为每类项目给出下一步建议。
5. 输出简洁的 triage 结果。

## 边界约束

- 不替团队拍板产品方向。
- 不在信息不足时伪造优先级结论。
- 不把临时分类结果包装成最终决策。

## 输出要求

- 给出清晰分类结果。
- 说明优先级判断依据。
- 标注仍需补充的信息。

# Pre-Publish Review

## 适用场景

- 已完成开发并准备发布
- 需要快速确认这次交付是否适合进入发布环节
- 需要一轮轻量但结构化的上线前检查

## 执行步骤

1. 检查本次变更范围是否和目标一致。
2. 检查验证命令是否已经真实运行。
3. 检查是否存在未处理的已知风险。
4. 检查是否有未提交、未推送或未说明的改动。
5. 输出简洁的发布前结论。

## 边界约束

- 不扩展成完整代码审查。
- 不重新设计方案。
- 不在没有证据的情况下声称“可以发布”。

## 输出要求

- 先给结论，再列阻塞项或风险项。
- 只指出真正影响发布判断的问题。
- 如果可发布，也要明确剩余注意事项。

# Work With PR

## 适用场景

- 需要理解一个 PR 改了什么
- 需要处理 review comments
- 需要判断 PR 是否还存在明显风险

## 执行步骤

1. 先梳理 PR 的目标和主要改动范围。
2. 分类 review comments，区分阻塞项和建议项。
3. 识别仍需确认的风险点。
4. 在授权范围内推进修复或整理结论。
5. 输出当前 PR 的状态判断。

## 边界约束

- 不替代最终合并决策。
- 不默认修改用户未授权的方向。
- 不把所有评论都等价视为必须改动。

## 输出要求

- 列出主要风险和待处理项。
- 说明哪些评论已处理，哪些仍待确认。
- 给出下一步建议。

# Security Research

## 适用场景

- 新增接口、登录、上传、外部集成等安全敏感功能
- 需要快速识别代码中的安全风险点
- 需要给出结构化的安全检查结果

## 执行步骤

1. 明确本次检查的代码边界。
2. 检查输入验证与输出处理。
3. 检查认证、授权和敏感数据流转。
4. 检查外部依赖、配置和潜在暴露面。
5. 输出风险项、影响范围和建议动作。

## 边界约束

- 不假装完成完整渗透测试。
- 不输出夸张或无法证实的安全结论。
- 对高风险判断要附带证据或明确前提。

## 输出要求

- 按风险等级组织结论。
- 每个风险项说明原因、影响和建议。
- 如果没有发现问题，也说明检查范围和剩余盲区。
